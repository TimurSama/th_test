# Настройка секрета TELEGRAM_BOT_TOKEN

## ⚠️ Важно: Добавьте секрет для работы GitHub Actions

Без секрета `TELEGRAM_BOT_TOKEN` сбор данных рынка не будет работать.

## Как добавить секрет

### Шаг 1: Откройте настройки репозитория

Перейдите по ссылке:
**https://github.com/TimurSama/th_demo/settings/secrets/actions**

Или вручную:
1. Откройте репозиторий: https://github.com/TimurSama/th_demo
2. Нажмите **Settings** (вверху справа)
3. В левом меню выберите **Secrets and variables** → **Actions**

### Шаг 2: Добавьте новый секрет

1. Нажмите **New repository secret**
2. Заполните форму:
   - **Name**: `TELEGRAM_BOT_TOKEN`
   - **Secret**: `8591869020:AAEUFxvA4uo5Ct1j62gtE39wI-gdkZfFRmw`
3. Нажмите **Add secret**

### Шаг 3: Проверка

После добавления секрета:
1. Перейдите в раздел **Actions**: https://github.com/TimurSama/th_demo/actions
2. Запустите workflow вручную: **Collect Market Data** → **Run workflow**
3. Workflow должен выполниться успешно

## Что делает секрет

Секрет `TELEGRAM_BOT_TOKEN` используется для:
- Инициализации базы данных (если требуется)
- Логирования и уведомлений (если будут добавлены в будущем)

**Примечание**: Токен бота нужен только для GitHub Actions. Для локального запуска используйте файл `.env`.

## Безопасность

- ✅ Секрет хранится в зашифрованном виде
- ✅ Виден только GitHub Actions
- ✅ Не отображается в логах
- ✅ Не попадает в код репозитория

## Troubleshooting

### Секрет добавлен, но workflow все равно падает?

1. Проверьте, что имя секрета точно: `TELEGRAM_BOT_TOKEN` (без пробелов)
2. Проверьте, что значение токена правильное
3. Попробуйте удалить и создать секрет заново

### Как проверить, что секрет установлен?

В логах workflow не должно быть предупреждения:
```
Warning: TELEGRAM_BOT_TOKEN secret is not set
```

Если видите это предупреждение - секрет не установлен или установлен неправильно.
